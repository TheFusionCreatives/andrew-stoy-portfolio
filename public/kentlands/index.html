The transformation

From a phone line with no memory
to a system that never forgets

When this work began, the practice ran on disconnected tools that captured nothing and carried no context. Three eras tell the story.

Era 0 — The void

A calling system with nothing synced. Calls came and went. No contacts, no history, no memory.

Era 1 — The patchwork

A spreadsheet for availability, Outlook for email, and a third-party call center relaying everything by hand. Three places, no connective tissue.

Era 2 — Today

One synced CRM, an automated matcher, self-updating clinician tools, and an automation engine working 24/7.

HIPAA, by design

Security wasn't bolted on — it's the foundation

Mental-health data is some of the most sensitive there is. Every tool was built so that protected information simply has nowhere unsafe to go.

🛡️ No patient PHI ever touches our servers

The public matcher uses a patient's criteria only to rank clinicians, then discards them. Nothing about the person is stored. The CRM — already HIPAA-covered — remains the single home for any real data.

✅ Allow-list, not block-list

Public tools can only ever see a tight set of explicitly approved fields. Add a new field in the CRM and it's hidden by default — the safe failure mode. Sensitive notes are never exposed.

🔑 Least-privilege by design

Public reads and clinician edits run through completely separate, narrowly-scoped paths — a breach of one can't touch the other. Credentials live only in encrypted infrastructure secrets, never in code.

🪪 Sessions that can only edit you

Clinicians are verified server-side, and each session is cryptographically bound to that clinician's own record — making it impossible to view or edit anyone else's profile.

⏱️ Brute-force protection

Sign-in is rate-limited, with automatic lockout after repeated failed attempts. A simple front door with a serious lock.

🔒 Locked-down access

Access is restricted to the real console domains only, and every write is independently re-checked on the server — the browser can never change a field it shouldn't.

A pre-launch readiness check confirms every clinician can sign in and flags any data gaps without ever exposing a single phone number or email — so the system goes live with confidence, not crossed fingers.
A person on the other end

Every step kept people in mind

The technology is in service of three people: a patient looking for help, a coordinator answering the call, and a clinician trying to do their work. We built for each of them with care.

🧭 For the patient — never a dead end

The matcher always returns someone, explains why they fit, and is honest about where they differ. A human coordinator is offered on every result. No empty screens, no jargon — you can describe how you feel and be understood.

☎️ For the coordinator — never lost

A guided console with a call flow for every situation, a values card on every call reminding them it's "a person, not a transaction," a glossary that defines terms on hover, warm pre-written messages, and a one-key path to crisis support when it matters most.

🩺 For the clinician — never anxious

One tap to change availability, a forgiving 4.8-second undo, an unsaved-changes guard, and a "what the matcher sees about you" card so they always know how they're being represented. Their data, in their hands.

♿ For everyone — accessible

Reduced-motion preferences respected, AA-contrast color, labeled controls for screen readers, full keyboard navigation, and layouts that work from a phone to a docked side panel.

LIAH Chat Support → CRM via API
